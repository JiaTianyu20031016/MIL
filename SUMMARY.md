# 实验记录和总结

本文件用于记录实验期间的主要发现、代码的主要更改过程、犯过的错误和经验教训、未来的前进方向等。

## 主要发现和实验结果

### 实验设置
我们主要在 Qwen3-4B 上开展实验。该模型具有一定的基础数学推理能力，并且参数量相对较小，因而适合作为基模型。

我们主要采用 PRM800K 和 Math-Shepherd 两个数据集作为训练数据。
1. 对于前者，我们亲自进行了数据处理，将数据组织成‘推理路径-推理步骤‘的两级形式。推理路径的正确性定义为：当且仅当**所有的推理步骤都正确**。由于 PRM800K 只标注第一个错误步骤的位置，对于该位置之后的步骤，我们将步骤级标签设置为 None （实际训练过程中，可以选择是否将这些步骤视为错误并纳入 loss 中）。由于 PRM800K 采用了主动标注策略，轨迹级类别非常不平衡：完全正确的轨迹仅占全部轨迹的 15% 左右。因此，我们对数据进行了筛选，构建了一个（轨迹层面）类别平衡的数据子集，称为 PRM800K-Balanced，包含大约 30，000 条轨迹。我们主要在 PRM800K-Balanced 上进行实验。
2. 对于后者，同样将数据组织成‘推理路径-推理步骤‘的两级形式。推理路径的正确性定义为：当且仅当**最后一个推理步骤正确**。由于 Math-Sheperd 的步骤级标签是利用 MC Rollout 自动标注的，因此最后一个推理步骤的正确性等价于最终推理结果的正确性。相比于 PRM800K 的路径级标签，Math-Shepherd 的路径级标签并不能完全指示路径上是否存在中间步骤错误：即使最终结果正确，也可能有潜在的过程错误。因此，相比于 PRM800K，Math-Shepherd 是一个更有挑战性（含噪声）的数据集。我们只在 Math-Shepherd 中取自 Math 的数据子集上进行训练，该数据子集的规模仍然远大于 PRM800K-Balanced。

我们主要采用 ProcessBench 作为测试基准，并用 F1 Score 作为主要的评价指标。该指标综合评估了 PRM 正确定位轨迹中首个错误步骤的能力。其取值在 0-100 之间。

### PRM800K-Balanced 上的实验结果
PRM800K 是一个人工标注的数据集，根据我们的数据处理流程，其轨迹级标签的含义是“轨迹中是否包含至少一个错误步骤“。该设定与 MIL 的设置完全一致。因此，通过在该数据集上进行 MIL，我们希望验证 MIL 方法在 PRM 训练中的有效性，而排除其他干扰因素。

在所有的实验中，我们将 Qwen3-4B 添加一个两层 MLP 作为分类头，构建一个判别式 PRM，作为学习器。
我们选用的训练参数如下：global batch size = 32，learning rate = 1e-5，gradient accumulation = 8。

我们测试了多种不同的 MIL 技术，包括 MinPool，AttentionPool，ConjecturePool，InstanceMeanPool 和 ProbMeanPool 等。我们对比的基线主要是：
1. 弱基线：仅取每条轨迹最后一个 token 处的 embedding 作为轨迹级表示，并在轨迹级标签上进行监督学习。这相当于训练一个 ORM，然后将 ORM 作为 PRM 使用。记作 ORM。
2. 强基线：取每个步骤最后一个 token 处的 embedding 作为步骤级表示，并在完整的步骤级标签上进行监督学习。这是一个完全监督的 PRM，我们认为它代表了 MIL 方法（弱监督方法）的能力上界。记作 FS-PRM。

实验主要有如下发现：
#### MinPool 是最有效的 MIL 方法。
MinPoolMIL 在测试集上表现出了非常有竞争力的性能。无论是在 PRM800K 自身的测试集上（IID 泛化），还是 ProcessBench 上（Non-IID 泛化），MinPoolMIL 都与 FS-PRM 表现出了不相上下的性能（甚至在PRM800K 测试集上还略优于后者）。**这证实了 MIL 方法在 PRM 学习中的有效性：可以在不利用步骤级标注的条件下实现与步骤级完全监督学习相同的性能。**

#### 基于 Attention 的 MIL 方法表现欠佳。
虽然大多数 MIL 文献都认为 AttentionPool 和 ConjecturePool 是最有效的 MIL 池化技术，但在我们的实验中，这两种方法的表现远不如 MinPool。

我们推测，这可能是由于我们的设置与其他常见的 MIL 场景（例如 WIS）有所不同：在我们的场景下，一个 bag （推理轨迹）中包含的 instance（步骤）数量相对较少，一般在几个到十几个之间。而传统 MIL 场景中，一个 bag 往往包含成百上千个 instance。这种情况下，就需要通过 attention 机制来从大量 instance 中选择出最有代表性的“key instance“，并忽略到大量潜在的无用 instance。在我们的场景下，一条轨迹中的步骤数量极其有限，因此无需通过 attention 进行实例筛选。与 attention pooling 相比，min pooling 包含了更强的约束，注入了更强的归纳偏置。

当然，我们目前实现的 attention pool 机制非常简单：我们采用了线性的 point-wise attention，既没有复杂的 attention score 计算网络，不同的 step 之间也没有通过 cross attention 来交换信息。*未来，我们可能会进一步优化相关的实现，来探索基于 attention pooling 的 MIL 方法的能力上界。*

#### ORM 泛化到 PRM 任务的能力比想象中要好。
我们发现，weak baseline（ORM）直接用作 PRM，其效果远比我们想象中好：在我们的参数设置下，ORM 在 ProcessBench 上的 F1 score 相比于 strong baseline （FS-PRM）并没有相差非常多（紧紧 7 个点）。

我们推测，这可能是因为：PRM800K 数据集的轨迹级标签表征的不仅是最终结果的正确性，更是直接指示了轨迹中是否存在过程错误。因此，在这样的标签上进行监督学习，促使 ORM 学会了判别轨迹内部是否存在错误，而不仅仅是判别最终答案的正确性。因此，ORM 的上述能力能够迁移到任意的轨迹前缀上，从而能够用作 PRM。

然而，我们也发现，ORM 到 PRM 的泛化能力是脆弱的，与训练参数显著相关。例如，当我们将学习率调整到 1e-6 时（保持其他所有条件不变），ORM 仍然能有效学习到轨迹级分类，但是其在首个错误检测任务中的 F1 score 在训练过程中迅速崩溃，最终值接近于 0。这可能是因为，较小的学习率使优化过程收敛到一个次优的局部最小值：仅判别最终答案正确与否，而不是判别轨迹中是否存在错误。但是这一假设尚未得到直接的验证，只是推测。*未来，我们可能需要进一步检查相关代码，保证实现的正确性，并开展进一步的实验来确认和分析这一有趣的现象。*

### Math-Shepherd 上的实验结果

Math-Shepherd 是用 MC Estimation 自动标注的数据集。根据我们的数据处理流程，其中轨迹级标签的含义是“最终结果是否正确“。相比于 PRM800K，Math-Shepherd 的步骤级标签和轨迹级标签质量都低很多，包含很多噪声（这是说，标签并不能完全指示“逻辑正确性“）。但是也正因如此，该数据集更加符合真实应用场景。通过在该数据集上进行 MIL，我们希望验证 MIL 方法在真实应用场景下，仅利用 rule-based outcome reward 学习 PRM 的能力。

在所有的实验中，我们将 Qwen3-4B 添加一个两层 MLP 作为分类头，构建一个判别式 PRM，作为学习器。
我们选用的训练参数如下：global batch size = 32，learning rate = 1e-6，gradient accumulation = 8。 由于Math-Shepherd 的数据量相对较大，我们选用了更小的学习率，一方面有助于训练稳定，同时也不会导致训练不充分。

以 PRM800K 上的实验结果为基础，我们主要测试 MinPool MIL 方法的效果。相似地，我们选用以下两个 baseline：
1. 弱基线：仅取每条轨迹最后一个 token 处的 embedding 作为轨迹级表示，并在轨迹级标签上进行监督学习。这相当于训练一个 ORM，然后将 ORM 作为 PRM 使用。记作 ORM。
2. 强基线：取每个步骤最后一个 token 处的 embedding 作为步骤级表示，并在完整的步骤级标签上进行监督学习。这是一个完全监督的 PRM，我们认为它代表了 MIL 方法（弱监督方法）的能力上界。记作 FS-PRM。

实验主要有以下发现：
#### MinPool 容易收敛到次优的局部最小值。
